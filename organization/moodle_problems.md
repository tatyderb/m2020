# Что не хватает в lms (moodle)
* Открытого языко-независимого API
	* сплошная ручная работа
* Чекер float с точностью
	* нужно переделывать задачи
* Конфигурация контрольной, конфигурация задачи в контрольной - характеристики задаются на уровне каждого варианта, а не задачи или контрольной
	* язык решения задачи (с, с++, python) даже для языко-независимых задач типа hello, world
	* количество открытых и закрытых тестов
	* valgrind добавлен во все задачи на c и c++
		* возрастает нагрузка на систему
* Языко-независимые задачи (очень сложно, языки вписываются в задачу, или 1 задача на язык)
	* нужно переделывать задачи
* высокая цена исправления ошибки в условии / тестах
	* нет операций над множеством задач
	* если тест начался, нельзя править задачи
* нет симлинков, только копии задач
	* нужно выпилить часть вариантов из задачи в тренировку или другой поток
*  вопросы судьям, ответы судей студенту и сообщения всем студентам
* антиплагиат
	* встроенного нет, есть скрипты Васюкова
	* нет возможности сравнить две посылки (тем более из разных КР)
	* нельзя фильтровать посылки по признакам (вариант 2 первой задачи, все ОК решения, посланные после 15:00), что сильно затрудняет ручную проверку на антиплагиат и процесс апелляции.
* версионность
* logins - во время КР, несмотря на напоминание, студенты забыли пароль от системы и брали телефон для восстановления пароля.
